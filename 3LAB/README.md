# Отчет по проекту: Анализ транзакций (JavaScript)

## Обзор кода
Предоставленный код представляет собой библиотеку вспомогательных функций для обработки массива объектов `Transaction`. Реализованы основные операции: фильтрация, агрегация (подсчет сумм и средних значений) и поиск.

### Обзор функций

### Функции вычисления
* **`getUniqueTransactionTypes`**: Собирает все типы транзакций и убирает дубликаты с помощью `Set`. На выходе — массив уникальных строк.
* **`calculateTotalAmount`**: Складывает суммы абсолютно всех транзакций в массиве, используя `reduce`.
* **`calculateTotalAmountByDate`**: Гибкий фильтр суммы. Считает общую стоимость только за конкретный год, месяц или день (в зависимости от переданных аргументов).
* **`calculateAverageTransactionAmount`**: Делит общую сумму на количество транзакций. Есть проверка на пустой массив для защиты от ошибки.
* **`calculateTotalDebitAmount`**: Выбирает только расходы (`debit`) и суммирует их.

### Функции фильтрации
* **`getTransactionByType`**: Возвращает новый массив, содержащий транзакции только указанного типа (`debit` или `credit`).
* **`getTransactionsInDateRange`**: Отбирает транзакции, дата которых попадает в заданный период «от и до».
* **`getTransactionsByMerchant`**: Фильтрует список по точному совпадению названия магазина или сервиса.
* **`getTransactionsByAmountRange`**: Возвращает транзакции, сумма которых находится в заданном диапазоне (например, от 50 до 100).
* **`getTransactionsBeforeDate`**: Отбирает все операции, совершенные строго раньше указанной даты.

### Функции анализа и поиска
* **`findMostTransactionsMonth`**: Анализирует даты и определяет месяц (в формате `YYYY-MM`), в котором было совершено больше всего операций.
* **`findMostDebitTransactionMonth`**: Работает так же, как предыдущая, но учитывает только дебетовые операции.
* **`mostTransactionTypes`**: Сравнивает количество дебетовых и кредитных операций и возвращает тип, которого больше (или строку `equal`).
* **`findTransactionById`**: Ищет в массиве одну конкретную транзакцию по её уникальному идентификатору.
* **`mapTransactionDescriptions`**: Создает массив, состоящий только из текстовых описаний транзакций (полезно для вывода списка покупок).

## Контрольные вопросы

### 1. Какие методы массивов можно использовать для обработки объектов в JavaScript?
Для обработки массивов объектов чаще всего используются следующие методы:
* `filter()`: Создание подмножества объектов по условию (например, только транзакции "debit").
* `map()`: Трансформация объектов или извлечение конкретных полей (например, получение массива всех ID).
* `reduce()`: Агрегация данных (подсчет общей суммы, группировка).
* `find()`: Поиск первого объекта, соответствующего критерию (например, поиск по ID).
* `some()` / `every()`: Проверка, соответствует ли хотя бы один или все объекты условию.
* `sort()`: Сортировка объектов по значению свойства (дате, сумме).
* `forEach()`: Выполнение побочных действий для каждого объекта.

### 2. Как сравнивать даты в строковом формате в JavaScript?
Существует два основных подхода:
1.  **Прямое сравнение строк (для формата ISO 8601):** Если строки имеют формат `YYYY-MM-DD`, их можно сравнивать как обычные строки (`'2019-01-01' < '2019-01-02'`), так как лексикографический порядок совпадает с хронологическим.
2.  **Преобразование в объект Date:** Приведение строк к объекту `new Date(string)` и сравнение через операторы `>`, `<` или метод `.getTime()`. Метод `.getTime()` возвращает количество миллисекунд, что удобно для точного сравнения.

### 3. В чем разница между map(), filter() и reduce() при работе с массивами объектов?

| Метод | Назначение | Результат |
| :--- | :--- | :--- |
| **`map()`** | **Трансформация.** Проходит по каждому элементу и изменяет его или вытаскивает часть данных. | Новый массив **той же длины**, что и исходный. |
| **`filter()`** | **Отбор.** Оставляет только те элементы, которые прошли проверку (вернули `true`). | Новый массив **той же или меньшей длины**. |
| **`reduce()`** | **Агрегация.** "Сжимает" массив до одного значения (число, строка, один объект или новый массив). | **Одно значение** любого типа. |
