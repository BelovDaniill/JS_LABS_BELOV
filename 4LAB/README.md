# Отчет по лабораторной работе номер 4

## Введение
**Цель работы:** Изучение и практическое применение различных способов создания объектов и реализации механизмов наследования в языке JavaScript.

В ходе выполнения работы были рассмотрены два основных подхода к организации кода в стиле ООП:
1.  **Современный синтаксис классов** (использование ключевых слов `class`, `extends` и `super`).
2.  **Классический подход через функции-конструкторы** (`prototype`).

## Реализация

#### 2.1. Реализация иерархии с использованием классов (ES6)
В первой части кода спроектирована иерархия игровых сущностей. 
*   **Базовый класс `Item`:** Инкапсулирует общие свойства (название, вес, редкость) и методы доступа (`getInfo`, `setWeight`). Реализована базовая валидация данных при изменении веса.
*   **Дочерний класс `Weapon`:** Расширяет базовый класс. Ключевой особенностью является вызов родительского конструктора через `super()` и переопределение (полиморфизм) метода `getInfo`. Добавлены специфичные для оружия методы: `use()` (симуляция износа) и `repair()` (восстановление состояния).

#### 2.2. Прототипное наследование через функции-конструкторы
Аналогичная логика реализована без использования классов:
*   Функция `ItemConstructor` определяет свойства экземпляра, а методы выносятся в объект `prototype` для оптимизации памяти.
*   В `WeaponConstructor` применено связывание контекста `this` с помощью `ItemConstructor.call`, что позволяет наследовать свойства.
*   Связь между прототипами установлена через `Object.create(ItemConstructor.prototype)`, что выстраивает цепочку наследования, позволяя методам `Weapon` иметь доступ к методам `Item`.

#### 2.3. Использование современных операторов и безопасность кода
В секции тестирования продемонстрированы механизмы безопасного программирования:
*   **Optional Chaining (`?.`):** Применен для предотвращения ошибок при обращении к свойствам или методам несуществующих объектов (`nothing?.getInfo?.()`).
*   **Nullish Coalescing (`??`):** В методе `use` функции-конструктора использован оператор для установки значения по умолчанию, что гарантирует корректную работу логики даже при отсутствии инициализированного поля `durability`.

## Выводы
Проделанная работа демонстрирует гибкость JavaScript в вопросах построения архитектуры приложений. Сочетание строгого структурирования классов с гибкостью прототипов позволяет создавать сложные системы с четкой логикой взаимодействия.

## Контрольные вопросы

### 1. Какое значение имеет this в методах класса?
Значение this в методах класса напрямую зависит от контекста вызова. Когда метод вызывается через экземпляр объекта, this ссылается на этот конкретный экземпляр, открывая доступ к его свойствам и другим методам

### 2. Как работает модификатор доступа # в JavaScript?
Модификатор # используется в JavaScript для объявления приватных полей и методов класса. В отличие от обычных свойств, которые доступны извне, к приватным членам можно обратиться только внутри тела самого класса.

### 3. В чём разница между классом и функцией-конструктором?

Класс нельзя вызвать без new. Если вызвать класс просто так, он выдаст ошибку, тогда как функция может создать неопределенное состояние.

В классах всегда включен строгий режим use strict. Это предотвращает создание глобальных переменных по ошибке и запрещает дублирование имен параметров в конструкторе.

Синтаксис: класс - это современный стандарт. Он заменяет ручное копирование прототипов встроенным механизмом extends, который корректно связывает родительский и дочерний классы одной строкой.
