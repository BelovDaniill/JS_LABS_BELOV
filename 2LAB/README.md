# Лабораторная работа №2: Реализация методов массивов

Данный проект содержит кастомную реализацию базовых методов работы с массивами в JavaScript без использования встроенных функций `Array.prototype`.

## Цель работы
Изучение принципов работы колбэк-функций и логики перебора массивов, обработка граничных случаев и типизация аргументов.

## Реализованные методы

### 1. forEach(array, callback)
Выполняет переданную функцию один раз для каждого элемента массива.
- **Возвращает:** `undefined`.
- **Особенности:** Строгая проверка на массив и функцию.

### 2. map(array, callback)
Создает новый массив с результатом вызова колбэка для каждого элемента.
- **Возвращает:** Новый массив той же длины.

### 3. filter(array, callback)
Создает новый массив со всеми элементами, прошедшими проверку в колбэке.
- **Возвращает:** Новый массив (длина <= исходной).

### 4. find(array, callback)
Возвращает значение первого элемента в массиве, который удовлетворяет условию.
- **Возвращает:** Элемент или `undefined`.

### 5. some(array, callback)
Проверяет, удовлетворяет ли хоть один элемент массива условию.
- **Возвращает:** `boolean`. Прерывает цикл при первом `true`.

### 6. every(array, callback)
Проверяет, все ли элементы массива удовлетворяют условию.
- **Возвращает:** `boolean`. Прерывает цикл при первом `false`.

### 7. reduce(array, callback, initialValue)
Применяет функцию-аккумулятор к каждому элементу массива (слева направо), сводя его к одному значению.
- **Особенности:** Если `initialValue` не указан, берет первый элемент массива как начальный.

## Ответы на контрольные вопросы

1. **В чем преимущества использования колбэков при работе с массивами?**
   Колбэки позволяют отделять логику *перебора* данных от логики *обработки* этих данных. Это делает код более универсальным, читаемым и позволяет переиспользовать одну и ту же функцию перебора для разных задач (фильтрация, трансформация и т.д.).

2. **Какие проблемы могут возникать при использовании колбэков и как их избежать?**
   - **Callback Hell:** Чрезмерная вложенность функций. Решается декомпозицией или использованием Promises/Async-Await (в более сложных задачах).
   - **Потеря контекста (this):** Решается использованием стрелочных функций или метода `.bind()`.
   - **Производительность:** Лишние вызовы функций на миллионах элементов могут быть медленнее обычного цикла `for`.

3. **Как реализовать функции без встроенных методов?**
   Реализация строится на использовании базового цикла `for (let i = 0; i < array.length; i++)`, ручном обращении к элементам по индексу `array[i]` и добавлении результатов в новый массив через `.push()` там, где это необходимо по условию метода.
